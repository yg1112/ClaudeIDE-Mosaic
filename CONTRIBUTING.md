# 如何为 Mosaic 贡献新资源

**让 Mosaic 的 UI 资源库更加丰富**

---

## 🎯 你可以贡献什么

### 1. 新的 UI 组件库
推荐你发现的优质开源组件库

### 2. 设计灵感平台
推荐新的设计资源和市场平台

### 3. 风格参考 App
推荐设计优秀的 App 作为参考

### 4. 设计系统模板
提供新的色彩系统、字体系统、间距系统模板

## 📝 如何添加新资源

### 方法 1：直接编辑 RESOURCES.md（最快）

打开 `RESOURCES.md` 文件，根据资源类型添加到对应部分：

#### 添加新的组件库

找到对应的表格（如"基础组件库"、"动效库"等），添加新行：

```markdown
| **库名称** | github.com/作者/项目名 | 风格标签 | 覆盖组件 | L1 |
```

**示例**：
```markdown
| **SwiftUIKit** | github.com/danielsaidi/SwiftUIKit | modern, native | Buttons, Cards, Lists | L1 |
```

#### 添加新的设计灵感平台

找到"设计灵感与市场平台"部分，添加到"灵感搜索平台"表格：

```markdown
| **平台名** | 网址 | 特点 | 使用场景 | Level |
|----------|------|------|---------|-------|
| **新平台** | example.com | 描述特点 | 何时使用 | L3 |
```

**你提供的示例（已添加）**：
```markdown
| **Mobbin** | mobbin.com | 真实App截图库，按流程分类 | 寻找成熟App的交互方案 | L4 |
| **Figma Community** | figma.com/community | 社区模板、插件、设计系统 | 寻找可直接使用的设计系统 | L3 |
| **UI8** | ui8.net | 付费/免费设计素材市场 | 高质量完整设计系统 | L3-L4 |
```

#### 添加新的风格参考 App

找到"风格参考库"部分，添加到对应风格分类：

```markdown
#### 🌙 Dark Mode / 高级感
- **新App名** - 描述其设计特点
```

**示例**：
```markdown
#### 🌙 Dark Mode / 高级感
- **Linear App** - 标杆级深色主题，紫色点缀
- **Raycast** - 命令行风格，极简
- **你的新App** - 你的描述
```

### 方法 2：使用 GitHub Issue（推荐给非技术用户）

如果你不熟悉编辑 Markdown 文件，可以：

1. 在 GitHub 上创建一个 Issue
2. 使用模板（见下方）
3. 维护者会审核并添加

**Issue 模板**：

```markdown
## 资源推荐

**资源类型**：[组件库 / 设计平台 / 参考App / 其他]

**名称**：[资源名称]

**链接**：[URL]

**推荐理由**：
[为什么你觉得这个资源值得添加到 Mosaic？]

**适用场景**：
[什么时候应该使用这个资源？]

**平台支持**：
[macOS / iOS / Web / 跨平台]

**资源级别**（如果是组件库）：
- [ ] L1 - 可直接使用（Swift Package / npm package）
- [ ] L2 - 需要配置或主题化
- [ ] L3 - 需要转译（React -> SwiftUI）
- [ ] L4 - 仅供参考（截图、设计稿）

**额外信息**：
[任何其他有用的信息]
```

## ✅ 资源质量标准

为了保证 Mosaic 的资源质量，请确保推荐的资源符合以下标准：

### 组件库标准

- ✅ **活跃维护**：最近 6 个月内有更新
- ✅ **文档完整**：有清晰的 README 和使用示例
- ✅ **质量可靠**：GitHub Stars > 50（或有其他质量保证）
- ✅ **易于集成**：提供 Swift Package / npm package
- ✅ **许可证友好**：MIT / Apache 2.0 等开源许可证

### 设计平台标准

- ✅ **内容丰富**：有足够多的高质量资源
- ✅ **易于访问**：不需要付费或复杂注册即可浏览
- ✅ **更新及时**：有持续的内容更新
- ✅ **可搜索**：支持关键词搜索或分类浏览

### 参考 App 标准

- ✅ **设计优秀**：在设计社区有认可度
- ✅ **代表性强**：代表某种典型的设计风格
- ✅ **可访问**：用户可以下载或体验

---

## 🏷️ 资源分级系统（Level）

| 级别 | 定义 | 典型例子 | Mosaic 处理方式 |
|------|------|---------|----------------|
| **L1** | 可直接使用的包 | Swift Package, npm package | 直接添加到依赖 |
| **L2** | 需要配置的库 | 需要主题化的组件 | 提供配置模板 |
| **L3** | 需要转译的资源 | React 组件 -> SwiftUI | 生成转译任务 |
| **L4** | 仅供参考 | 截图、设计稿 | 生成像素级复刻任务 |

## 🔍 如何发现好资源

### 1. GitHub 搜索技巧

```
# SwiftUI 组件库
"swiftui components" stars:>100 pushed:>2024-01-01

# React 组件库
"react ui library" stars:>1000

# 设计系统
"design system" OR "design tokens"
```

### 2. 设计平台推荐

- **Mobbin** - 真实 App UI 截图
- **Figma Community** - 设计系统和模板
- **UI8** - 付费/免费设计素材
- **Dribbble** - 设计师作品展示
- **Behance** - 完整案例研究
- **ProductHunt** - 发现新的设计工具

### 3. 关注设计博客和社区

- **Apple Design Resources** - Apple 官方设计资源
- **Material Design** - Google 设计规范
- **Refactoring UI** - 实用设计技巧
- **Laws of UX** - UX 设计原则

## 📊 资源审核流程

1. **提交**：通过编辑文件、Issue 或 PR 提交资源
2. **审核**：维护者检查资源质量和分类
3. **测试**：（如果是组件库）测试是否可正常使用
4. **添加**：合并到 RESOURCES.md
5. **更新**：更新 Mosaic 的搜索索引

## 💡 贡献示例

### 示例 1：你发现了一个优秀的 SwiftUI 动效库

1. 打开 `RESOURCES.md`
2. 找到"动效库"部分
3. 添加新行：

```markdown
| **YourAnimationLib** | github.com/author/lib | 丝滑转场动效 | 页面切换、状态变化 | L1 |
```

4. 保存并提交

### 示例 2：你发现了一个新的设计灵感网站

1. 打开 `RESOURCES.md`
2. 找到"设计灵感与市场平台" → "灵感搜索平台"
3. 添加新行：

```markdown
| **DesignInspire** | designinspire.com | 设计案例精选 | 寻找特定行业的设计灵感 | L4 |
```

4. 保存并提交

### 示例 3：你想推荐一个设计很棒的 App

1. 打开 `RESOURCES.md`
2. 找到"风格参考库" → 选择合适的分类
3. 添加：

```markdown
#### ☀️ Light Mode / 清爽感
- **Bear** - 极简写作风
- **Things 3** - 精致细节
- **YourApp** - 简洁优雅的笔记应用，留白充足
```

4. 保存并提交
