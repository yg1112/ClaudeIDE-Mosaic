### 2. 增强版问诊流程

**从机械问卷升级为设计咨询对话！**

#### 核心改进：

**a) 深度互动问题**
- 不只收集信息，更提供专业见解
- 每个问题基于项目实际情况
- 问题后附带专家观察和建议

**示例对比**：

v1.0（旧）：
```
Q: 你希望用户打开软件时的第一感觉是什么？
[选项 1/2/3/4]
```

v2.0（新）：
```
Q: 我看到你的项目主要是 macOS 生产力工具，
   通常这类工具有两种设计方向：

   1️⃣ 工具感强（像 VS Code、Linear）
      - 深色背景，减少视觉疲劳
      - 信息密度高，专业高效
      - 快捷键优先

   2️⃣ 友好易用（像 Notion、Slack）
      - 浅色柔和，视觉舒适
      - 留白充足，降低学习门槛
      - 交互引导明确

   你希望偏向哪个方向？或者有第三种想法？

   💡 我的观察：考虑到你的用户是开发者，
   他们可能更习惯深色主题...
```

**b) 实时网络搜索**

当用户提到参考 App 时，**Mosaic 会立即搜索并分析**：

```
你: 我喜欢 Linear 和 Raycast

Mosaic: 好的，让我快速研究一下这些 App...
        [使用 WebSearch 搜索]

        我明白了！你喜欢的这两个 App 有这些共同点：
        - 深色主题，减少视觉疲劳
        - 极简设计，没有多余元素
        - 快捷键优先，专业高效
        - 微妙的紫色/红色点缀

        💡 所以我推测你喜欢"专业工具感"的设计，对吗？
```

**c) 追问深入**

不只问表面问题，更要帮用户想到他们没想到的：

```
Q: 你的用户通常在什么场景下使用软件？
   他们每天用多久？
   他们同时还会用什么其他软件？

💡 为什么我问这个：不同使用时长和场景，对颜色、
   对比度、动效的需求完全不同。整天盯着看的软件
   需要更柔和的配色。
```

**d) 问题列表升级**

新增问题：
- Q0: 验证项目探索的准确性
- Q5: 优先级确认（哪些功能重点设计）
- Q6: 限制和禁忌

总问题数：从 5 个增加到 7 个（但更有针对性）

## 🔧 技术改进

### 工具使用

Mosaic 现在充分利用 Claude Code 的工具：

- **Glob** - 项目文件结构探索
- **Grep** - 代码搜索和分析
- **Read** - 读取配置和文档
- **WebSearch** - 实时搜索 GitHub 和设计平台
- **WebFetch** - 获取资源详情

### 工作流程升级

```
v1.0 (4 个阶段):
  问诊 → 采购 → 选型 → 交付

v2.0 (5 个阶段):
  项目探索 → 问诊 → 采购 → 选型 → 交付
  ↑ 新增      ↑ 增强   ↑ 增强
```

## 💡 核心理念升级

### v1.0 的理念：

1. 用户只做 CEO，不做设计师
2. 采购优先，不造轮子
3. 中间层解耦
4. 端到端可验收

### v2.0 新增理念：

5. **先看代码，再问问题** - 项目感知优先
6. **审美要比用户更懂** - 提供专业建议
7. **发现用户没想到的** - 主动发现设计机会
8. **互动而非问卷** - 设计咨询对话
